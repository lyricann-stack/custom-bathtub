# 3D 工具調研報告 — 客製浴缸 3D 自動化

> 全球客製浴缸專案・前置作業清單 D 項交付物
> 調研日期：2026-06-12｜調研：Lyric ＋ AI

---

## 0. 需求回顧（出自 6/12 與 Masa 會議）

| # | 硬性需求 | 備註 |
|---|---|---|
| 1 | 2D（手繪草圖／照片／平面圖）→ 3D | 客戶端輸入多樣 |
| 2 | 指令式／參數化修改（曲線改直線、R 角角度），非整個重生成 | 阿中下術語、AI 執行 |
| 3 | 有 API 或可程式化，能被 AI Agent 呼叫 | 24/7 工作流的前提 |
| 4 | 曲面品質適合浴缸（光滑曲面，製造級 NURBS 佳） | 上次用 Claude 直做 2D→3D「慘不忍睹」＝工具不對 |
| 5 | 中國境內可用（阿中在東莞） | 防火牆、本土替代品 |

**目標**：阿中單件修改 1 小時 → 3–5 分鐘。

---

## 1. 核心結論（先講答案）

**沒有單一工具能包辦，要用「雙路徑管線」：**

```
客戶草圖/照片 ──→ 【路徑A：概念展示】AI 生成 mesh（秒級出圖給客戶確認）
                      Tripo / Hunyuan3D / Rodin
        確認後 ──→ 【路徑B：製造修改】Rhino（NURBS）＋ MCP 讓 Claude 直接操作
                      阿中下術語 → Agent 改參數 → 出 STEP/IGES 給工廠
```

- **路徑 A（給客人看）**：AI mesh 生成工具又快又便宜，但輸出是 mesh（三角網格），**不能直接拿去製造**——這正是上次「慘不忍睹」的原因之一：拿概念工具當製造工具用。
- **路徑 B（給工廠用）**：浴缸是光滑自由曲面，製造級修改的正解是 **Rhino（NURBS 原生）**，而且 2025–2026 已出現多個 **Rhino/Grasshopper MCC（MCP）開源橋接**，Claude Agent 可以直接在 Rhino 裡建模、改曲線、改 R 角——「阿中術語 → AI 執行」技術上已經成立。
- **2,000+ 款資料庫的隱藏紅利**：常見形體做成 Grasshopper **參數化模板**後，改曲線／R 角＝調一個參數，連 AI 都不用等，秒改。

## 2. 候選工具總覽

### A 類｜AI 生成 3D（影像/草圖 → mesh）— 路徑 A 用

| 工具 | 公司/背景 | API | 價格概況 | 強項 | 弱項 |
|---|---|---|---|---|---|
| **Tripo AI** | VAST（中國團隊） | ✅ $0.01/credit，註冊送 2,000 | v3.0 有 **sketch-to-3D** 專用管線、Ultra 模式 200 萬面、Magic Brush 2.0 局部修補、智慧部件分割 | 形狀準確度社群評價最高之一；速度快 | mesh 輸出，非製造級 |
| **Rodin（Hyper3D）** | Deemos（上海，上科大孵化） | ✅ Business 方案 $120/月含完整 API | Gen-2 100 億參數、Gen-2.5（2025/11）數千萬面 ~4 秒生成；表面寫實度市場最佳 | 品質天花板；中國公司、境內可用性佳 | 最貴；mesh 輸出 |
| **Hunyuan3D** | 騰訊 | ✅ 開源可自架＋第三方 API（~$0.25/次） | **開源免費可商用**、可自架在東莞/泰國伺服器；標準版 ~45 秒生成 | **中國境內零障礙**、零訂閱成本、資料不出門 | 需自備 GPU；品質略遜 Rodin |
| TRELLIS / TRELLIS-2 | Microsoft（開源） | 自架 | 免費 | 視覺擬真度最佳（社群評價） | 同上，mesh |
| Meshy | — | ✅ $10–30/月 | 3D 列印工作流佳、Blender/Unity 插件 | 對我們場景無突出優勢 | mesh |

### B 類｜Text-to-CAD（文字 → 參數化 CAD）

| 工具 | API | 輸出 | 評估 |
|---|---|---|---|
| **Zoo（前 KittyCAD）** | ✅ 開源、ML-ephant API | **STEP 為原生真相來源**，KCL 語言可逐步編輯 | 參數化＋STEP 很對路，但 KCL 擅長機械稜柱類零件；**浴缸自由曲面是其弱項**，列為觀察名單 |
| Adam / Leo AI 等 | 部分 | 多為機械件 | 同上，暫不入圍 |

### C 類｜Mesh → CAD 重建（路徑 A 接路徑 B 的橋）

| 工具 | 能力 | 評估 |
|---|---|---|
| **Backflip AI** | 掃描/草圖/圖片 → **可製造 STEP**；訓練於一億+幾何資料集；相容 Onshape/STEP/STL/OBJ | AI mesh 自動轉 CAD 的最佳候選，PoC 必測 |
| QUICKSURFACE | 傳統逆向工程：自由曲面 NURBS 擬合 | 人工操作為主，備案 |

### D 類｜可程式化 CAD ＋ AI Agent 操控（路徑 B 核心）

| 工具 | 角色 | 評估 |
|---|---|---|
| **Rhino + Grasshopper** | NURBS 原生建模，工業設計標準，中國使用者基數大、買斷制授權 | **路徑 B 主力**。浴缸曲面＝NURBS 主場；輸出 STEP/IGES 直通模具 |
| **Rhino/GH 的 MCP 橋接**（開源，2025–2026 湧現） | 讓 Claude 直接操作 Rhino/Grasshopper | 候選：[RhinoMCP（jingcheng-chen）](https://github.com/jingcheng-chen/rhinomcp)、[grasshopper-mcp（alfredatnycu）](https://github.com/alfredatnycu/grasshopper-mcp)、[Cordyceps](https://github.com/brookstalley/cordyceps)、[GH_mcp_server](https://github.com/veoery/GH_mcp_server) |
| **Ant**（food4Rhino，2026/2 上架） | Grasshopper 內的 AI copilot，能理解現有定義並修改 | 阿中端的「副駕駛」候選，降低他的學習門檻 |
| Raven / Smarthopper | 自然語言＋參考圖生成 GH 腳本／技術除錯 | 備選 |
| Blender Python | 免費可程式化 | mesh 為主，製造級曲面不如 Rhino，列備案 |
| Onshape / Fusion 360 API | 雲端 CAD API | 偏機械件、雲端在中國連線不穩，暫不入圍 |

## 3. 入圍名單（4 個）＋建議管線

### 入圍
1. **Tripo AI**（路徑 A 首選：sketch-to-3D 專用管線＋API 便宜）
2. **Hunyuan3D 自架**（路徑 A 中國備援＋長期成本最低，東莞/泰國伺服器可自架）
3. **Backflip AI**（橋：mesh → STEP）
4. **Rhino + Grasshopper + MCP 橋接**（路徑 B 核心：阿中術語 → Agent 改 NURBS → 出 STEP）

> Rodin 列「品質升級備選」：若 Tripo 形狀準確度不夠再上，$120/月。

### 建議管線全圖

```
① 接洽 Agent 收到客戶草圖/照片
② Tripo sketch-to-3D（或 Hunyuan3D）→ 3D 概念圖 → 渲染給客戶
③ 客戶確認方向 → 進入客製修改：
   a. 資料庫有相近形體 → 調出 Grasshopper 參數化模板 → 改參數（秒級）
   b. 全新形體 → Backflip mesh→STEP → Rhino 重建 NURBS
④ 阿中下術語指令（「這條改 R50」「直線變曲線」）
   → Claude 經 RhinoMCP 執行 → 阿中看結果點頭
⑤ 來回 2–3 次定案 → 出 STEP/IGES → 接 PI 報價流程
```

### 成本估算（啟動）

| 項目 | 費用 |
|---|---|
| Tripo API | 註冊送 2,000 credits，後續 $0.01/credit（PoC 近乎免費） |
| Hunyuan3D | 開源免費（需一台 GPU 機，可先用現有設備或雲端按時計費） |
| Backflip AI | 訂閱制，PoC 期間單月即可 |
| Rhino 授權 | 買斷約 US$995/席（阿中若已有 Rhino 則為零） |
| MCP 橋接 | 開源免費 |

### 中國境內注意事項（阿中端）
- **Hunyuan3D（騰訊）境內零障礙**；Tripo、Rodin 皆中國團隊，境內可用性佳。
- Rhino 為本機軟體，無防火牆問題。
- **Claude 在中國境內存取受限**：建議 Agent 跑在你（澳洲）或泰國側，透過遠端連線操作阿中機器上的 Rhino MCP，或阿中端走公司 VPN——此項在 PoC 中一併驗證。

## 4. PoC 測試計畫（兩週）

**驗收標準（會議定的）**：3–5 個真實浴缸款式端到端跑通，產出「看得出是浴缸」且可交付的 3D；單件修改時間壓進 3–5 分鐘。

### 第 1 週｜分頭測試
- [ ] 選 5 個真實款式：3 個常規（資料庫內有 3D）＋ 1 個手繪草圖 ＋ 1 個僅照片
- [ ] **路徑 A 對比**：同一組輸入分別餵 Tripo / Hunyuan3D（/必要時 Rodin），按「形狀準確度／曲面光滑度／生成速度／成本」打分
- [ ] **路徑 B 架設**：裝 Rhino ＋ 測 2 個 MCP 橋接（RhinoMCP、grasshopper-mcp），挑 1 個浴缸建成 Grasshopper 參數化模板
- [ ] 用自然語言下 10 條修改指令（改 R 角、改曲線、改長寬深）驗證執行準確率

### 第 2 週｜端到端串接
- [ ] 完整跑一次：草圖 → AI mesh 概念圖 → （Backflip 轉 STEP）→ Rhino 修改 → 出 STEP
- [ ] 請**阿中遠端參與一次修改**：他口述術語、Agent 執行，計時對比現行 1 小時基準
- [ ] 驗證中國連線方案（遠端操作 vs VPN）
- [ ] 產出工具定版決議 ＋ 各工具費用訂閱清單 → 給 Masa 過目

### 風險與備案
| 風險 | 備案 |
|---|---|
| AI mesh 形狀不準（草圖太抽象） | 加一步「AI 先生成多視角概念圖讓客戶選」再轉 3D |
| Backflip 轉 STEP 曲面品質不足 | 退回 Rhino 人工重建（阿中 15–20 分鐘）＋ QUICKSURFACE |
| MCP 橋接不穩定（開源早期專案） | 四個候選互為備援；最壞情況用 Rhino Python 腳本庫＋Ant copilot |
| 阿中接受度 | 強調「他仍是拍板者」，AI 只是手；先從他最煩的重複修改下手 |
